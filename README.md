# 硕士研究生学位论文开题报告

## 一、 论文选题依据

### （一） 选题背景

随着 5G 网络规模化部署，其在高带宽、低时延、广连接等方面的能力已得到广泛验证。然而，在复杂 5G 网络环境中（如边防哨所、近海港口、人流密集场所等），仍存在诸多挑战。这些区域普遍存在信号覆盖弱、链路不稳定、干扰强烈等问题，导致传统单链路传输机制难以同时满足高可靠性、低时延与高带宽利用率的综合需求。

随着云遥操作、无人系统协同、高清视频回传、AR/VR 远程巡检等新兴业务的发展，这些复杂场景对传输质量提出了更高要求。传统单路径传输机制难以兼顾鲁棒性与实时性，无法满足新兴业务的传输需求。

**多路径汇聚（Multipath Aggregation）** 技术通过并行利用多条传输路径实现带宽叠加与路径容错，是提升传输可靠性的有效手段 [1]。但现有方案（如 MPTCP、MP-QUIC）在面对随机丢包与相关丢包时，多采用固定路径冗余或单路径重传，导致带宽利用率不足 [2][3]。

与此同时，**轻量级前向纠错（FEC）** 技术可在毫秒级时间内修复随机丢包，具备低时延优势。然而，传统 FEC 机制通常面临高冗余率与低时延的权衡困境：冗余过高则浪费带宽，冗余不足则恢复失败率高。

因此，针对复杂 5G 网络环境下的易干扰、易丢包和易抖动等特性，如何实现稳定、可靠、高效的实时海量数据传输，成为当前学术研究与产业部署的关键技术问题。多路径调度与轻量级 FEC 的联合优化与动态融合可以较好地解决前述问题。

### （二） 研究意义

本研究旨在建立“多路径汇聚 + FEC 补偿”融合优化模型，通过凸优化刻画冗余度与有效码率、时延之间的定量关系，为多路径与丢包补偿的协同优化提供理论依据。

1. **算法设计**：设计一种基于在线凸优化（OCO）的动态融合算法，在毫秒级时间内自适应调整路径冗余与 FEC 强度，实现“按需冗余 + 智能补偿”。
2. **应用场景**：成果可应用于边防哨所、近海港口、交通枢纽、工业园区等 5G 特定场景。
3. **价值体现**：为复杂 5G 网络环境提供低时延、高可靠的传输方案，降低传输中断风险，提升带宽利用率，满足实时交互类应用的响应需求，具备工程推广及产业落地价值。

### （三） 国内外研究现状

1. **多路径汇聚**

   - Paasch 等 [1] (2014) 在 RFC 6824 中实现内核级 MPTCP，奠定包级调度基础。
   - Google 推出的 MP-QUIC (Iyengar, 2023) 将多路径能力上移至用户态，支持帧级乱序控制，但两条路径的冗余比例固定，无法根据实时丢包特征自适应调整，导致带宽利用率下降 [4]。

2. **丢包补偿**

   - 现有方案在单路径场景下呈现出“时延-开销-算力”三角矛盾。
   - XOR-FEC+NACK 方案恢复时延（40–60 ms）高于交互业务上限；RS(8,6) 时延约 10 ms 但带来 33% 带宽开销 [5]。
   - AI-PLC (Zhang, 2022) 利用 GAN 做帧级修复，冗余低于 5% 但计算量巨大，难以在终端落地 [6]。现有补偿机制多面向单路径，且未与多路径调度联动设计。

3. **融合机制探索**

   - MIT CSAIL (2021) 在 MPTCP 层集成 RS 编码，将丢包率从 2% 降至 0.1%，但未考虑门限自适应 [7]。
   - 近年研究（如 LDMP-FEC、HOP、Peekaboo 等）提出“轻量 FEC + MPTCP 包级调度”思路 [8]，但多依赖经验参数，缺乏统一理论模型和动态优化机制。

4. 总结

   目前多采用固定冗余或单路径补偿，无法在毫秒级动态调整。理论模型缺失与实时决策机制缺位，使得协同增益未释放。本文旨在构建融合优化框架，填补这一空白。

## 二、 研究方案

### （一） 研究目标

面向复杂 5G 网络的链路抖动与突发丢包挑战，研究多路径汇聚技术和轻量化 FEC 技术，构建“多路径 QUIC + 轻量级 FEC”融合框架，实现动态融合系统，提供稳健、实时、高效的传输支撑。

### （二） 研究内容

1. **性能边界建模**：建立多路径冗余度与 FEC 冗余的联合优化模型，推导有效码率、丢包率与时延的定量关系，刻画 Pareto Frontier。
2. **在线自适应融合算法研究**：基于 Online Convex Optimization (OCO) 框架，实时监测链路特征，实现毫秒级自适应调整。
3. **轻量级 FEC 模块设计与优化**：利用 SIMD 与 ISA-L 库进行编码加速，优化 RS(10,8) 或 Fountain 编码。
4. **系统集成与原型实现**：在 MP-QUIC 协议栈中集成 OCO 调度器与轻量级 FEC 模块。
5. **实验验证与性能评估**：构建测试平台，开展高清视频流和大文件传输实验，对比分析关键指标。

### （三） 拟解决的关键问题

1. **性能边界刻画**：推导有效码率–时延–冗余度的定量关系，分析协同增益理论上界。
2. **动态融合优化机制**：基于 OCO 与丢包统计特征，设计自适应门限触发机制，实现“按需冗余+智能补偿”。

### （四） 研究思路与方法

- 构建统一凸优化模型，给出性能边界。
- 采用 OCO 实现毫秒级动态调度。
- 在 `liblsquic` 中嵌入 OCO 调度器和 ISA-L 加速的轻量 FEC。
- 搭建测试平台，实测数据回灌模型，驱动算法持续优化。

### （五） 实验方案与可行性

- **硬件平台**：ARM 开发板（发送端）+ PC（接收端）+ 5G 网卡 + 网络损伤仪。
- **软件栈**：MP-QUIC 使用 `liblsquic` 库，FEC 使用 Intel `ISA-L` 库。
- **测试内容**：通过高清视频流（1080p/60fps）和大文件传输，测量平均时延、95 分位时延、丢包恢复率等。

### （六） 研究特色与创新点

1. **统一优化模型**：将多路径冗余度与 FEC 编码强度纳入统一凸优化框架，揭示冗余开销与传输时延的定量权衡关系。
2. **相关性门限触发机制**：针对丢包相关性特征，提出相关性门限触发机制（$\rho > \rho_0$），实现动态控制，在极端条件下保持高可靠与低时延。
